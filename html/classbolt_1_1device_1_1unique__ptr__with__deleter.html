Detailed Description

template<class TType, class TDeleter>
class bolt::device::unique_ptr_with_deleter< TType, TDeleter >

unique_ptr equivalent for device pointers. Its primary purpose is to enable transparent resource management on the same level as host unique_ptr. First incentive for its introduction was to avoid move constructors in DeviceImage class while still allowing it to be stored in STL containers Naming note: It intentionally follows the naming scheme of unique_ptr instead of BoltView's naming conventions to emphasize the equivalent meaning and usage
